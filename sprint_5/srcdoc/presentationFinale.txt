== Bilan fonctionnel

=== OPTI

-  Afficher des informations à propos des concepteurs
-  Créer/Modifier/Supprimer un sujet, un étudiant, un projet, un intervenant, un groupe
-  Associer des sujets à des gropues de projet, un étudiant à un groupe
-  Lister les projets (non) affectés et les équipes avec ou sans projet
-  Ouvrir des fichiers CSV
-  Sauvegarder dans des fichiers CSV

Les autres fonctionalités n'étant pas prêtes, nous ne sommes pas en mesure de vous les présenter.

=== OPTI Web

-  Afficher les données des étudiants, des projets, des intervenants et des sujets
-  Toutes les données sont croisées

NOTE: Tout le backlog du produit OPTI Web a été respecté.

== Bilan technique

-  Toutes les fonctionalités n'ont pas été implémentées dans l'application finale.
-  Nous avons manqué de temps et d'organisation pour mener à bien le projet.
-  Certaines personnes étaient complètement en dehors du code
-  Au contraire, certains membres étaient irremplaçables sur leurs portions de code

== Bilan humain

-  La cohésion aurait pu être meilleure au sein du groupe
-  L'organisation aurait du être meilleure
-  La pression de la méthode Agile aurait pu être mieux gérée

== Bilan général

Nos avons appris beaucoup de choses, malgré l'échec partiel du projet :

-  Nous avons eu une vision de la méthode Agile
-  Nous avons eu une première expérience sérieuse de travail dans un groupe de projet
-  Nous avons appris l'utilisation de divers outils de gestion de projet : Redmine, GitHub, Travis, Subversion
-  Nous avons appris l'utilisation de divers outils de tests : Casper, Phantom
-  Nous avons appris l'utilisation de divers outils de génération de documents : ASCIIdoc, PlantUML
-  Nous avons appris à réaliser une application avec le modèle MVC
